Capstone Project Descriptions

Orchestrate: Automating Network Administration

Small businesses rely on efficient network infrastructure to operate and compete effectively. However, managing network devices, such as routers and switches, can be a daunting task, particularly in environments where dedicated IT specialists are scarce or unavailable. Orchestrate is a comprehensive network automation tool designed to address this challenge by combining the power of Python backend scripting with a user-friendly Svelte web interface. Orchestrate automates the discovery and configuration of network devices using Python’s powerful networking libraries, making network management more accessible and less time-consuming. It then offers intelligent default configurations, streamlining the often complex and error-prone setup process. Orchestrate helps small businesses save time and enhances network security and reliability.

🚗 Merrimon Avenue Traffic Simulation

Traffic simulations are useful when assessing potential changes to a road system. As a major arterial road, Merrimon Avenue needs to be able to handle a large traffic capacity while maintaining a consistent speed. However, it has experienced immense traffic congestion since a renovation in 2022. An ideal solution to this problem would be changing the road system to one that is more suitable to the amount of traffic without creating an unsafe environment for drivers or pedestrians. To achieve this, we have created a simulation in Unreal Engine 5 to compare different road systems and optimize traffic flow, congestion, and safety in a controlled virtual environment. This simulation is a valuable tool for understanding and improving traffic in UNC Asheville’s local community.

Multiplayer video game

Tower Defense (TD) is a well-established video game genre. While many TD games involve a multiplayer component, it is typically symmetric: every player has the same role. This project revitalizes the genre through a unique asymmetric multiplayer mode. One player engages in a mostly traditional TD game attempting to defend a spire. Opposing players attack the spire using their skill and intuition, alongside AI agents, to avoid the defenders’ traps in pursuit of destruction. The defending player must use their scarce resources wisely to confuse and destroy the hoard of NPC and player attackers. This and supporting game elements come together to create a fast-paced asymmetrical experience where all parties utilize TD game mechanics and real-time strategy.

Pindie

Social media has significantly changed the landscape of the internet and enhanced networking; however, there are many areas where it lacks. The social media platform, “Pindie”, was designed with an emphasis on collaboration and improvement in various creative fields. This platform allows independent artists to “pin” their creations and gather feedback from an audience. A user’s profile acts as a digital portfolio, allowing individuals to expand their audience through a built-in promotion system. Pindie allows multiple users to collaborate and author a singular pin through the authoring system. The front end was developed using the React framework, and allows users to create pins using a range of compatible mediums. The back end consists of an Express API made with Node.js and a MySQL database. Pindie, as a full stack web application, was tailor made for independent creators to improve, share, and network.

Bent Creek Buddy

Mountain biking is a popular recreational activity in Asheville, North Carolina. Consistent monitoring and reporting of trail conditions helps to ensure user safety and enjoyment of the local trail systems. However, there is no centralized platform for the community to share and access trail condition reports which is problematic for effective planning and decision-making. Bent Creek Buddy, a user-friendly app developed using JavaScript and React-Native, addresses this gap by facilitating user generated reports on important information such as recent trail conditions, potential obstacles, or trail damage. This app relies on the Express framework and an SQL-based relational database, which allows users to input both positive and negative trail conditions, creating a collaborative environment for maintaining trail safety and longevity. Users can check or submit reports on specific trails, receive condition reports, and are notified of any urgent flagged trails on their planned routes. Bent Creek Buddy not only contributes to the mountain biking world by offering real time data but also promotes community engagement and a sense of collective responsibility for trail maintenance and safety in our area.

Quasar Absorption Lines Database

Simulations of the cosmos indicate that the flows of gas into and out of galaxies greatly influence the changes in our universe. While these gases are not visible with standard telescopes, by using distant quasars as a backlight, these gases and their properties can be studied. These properties are measured through distortions found in the light emitted by the quasars and recorded in the tens of thousands of readings, known as quasar absorption lines. Due to the abundance of data contained in these files, it can be complicated for most people to utilize the data. This project seeks to resolve the accessibility problems for users looking to access this information. We present an Angular web application with a simple interface to view and interact with the quasar measurements. Besides the quasar data catalog, additional documentation is available for these measurements should the user request it. A web API using C# supports the flow of data for the application. This application provides a user-friendly interface for interacting with this data for people wanting to conduct astronomy research.

Small Timescale to Large Timescale Exoplanet Modeler and Plotter

The Large Synoptic Survey Telescope (LSST) conducts extensive sky surveys, producing an image every three days over a decade, serving as a cornerstone in astronomical research. Smaller telescopes, on the other hand, capture multiple images in much shorter durations. Given the rapid and frequent observations of these smaller telescopes, assuring the accuracy of their findings is a considerable challenge. We present a software package that uses LSST’s long-term images to corroborate the short-term observations from smaller telescopes, with a primary focus on the confirmation of exoplanets. We have created an interactive plot for viewing magnitude changes in a star in both the small-time scales and large-time scales. Additionally, the package will be able to generate simulated large-time scale images from LSST from other images. Because of the periodic nature of transiting exoplanets, observing a star on both the small-time scale and the large-time scale adds a means of confirmation. This enhancement in the validation process is critical, adding a substantial layer of confidence to exoplanet observations and is hoped to facilitate the confirmation of more exoplanets and foster the exploration of the uncharted.

Pyile: Peer to Peer Reimagined

Digital communication is a part of everyday life. From text messages containing private conversations to bank transactions, it is vital that communication is robust, private, and secure. However, many digital platforms are lacking in at least one of these areas. Pyile is a new Python library developed for secure peer-to-peer communication. The Pyile protocol uses a central authenticating peer, providing the reliability of a client-server relationship while preserving the privacy of a peer-to-peer network. The architecture allows the network to scale to accommodate a large number of peers, and the protocol ensures that it is impossible for any external entity to read or intercept messages. The utility of the Pyile protocol is demonstrated through a Python chat application.

Restricted Pixels: a Pixel-Art Web Game

"Restricted Pixels" is a web game that encourages the development of creative and problem-solving skills through a familiar game format. One player is designated as the “artist” and is tasked with drawing a randomly selected word while the other players, the “guessers”, must guess what the word is based on the pixel art drawing. The challenge for the artist is that they must work with a small, randomly selected palette of colors and restricted canvas size. Players are awarded points based on both their drawing skills and guessing skills, which are also scaled based on the difficulty chosen by the artist each turn. The back-end code uses node.js and the sockets.io library for responsive, real-time client-server communication. For the front-end, the code uses the p5.js library to track mouse position of the artist and to draw the pixels on the canvas. "Restricted Pixels" combines all these elements together to create a fast-paced, multiplayer game that is both challenging and fun to play. Link Here!
